# Step 1.1 — Batched SRS Generation (Sonnet 4.6 Compatible)

## Why Batched?

The original single-prompt SRS generation exhausted Opus quota before producing output because
the task (read full RBS → write 50–100 page SRS) exceeds a single Claude session's practical
output capacity. The solution: split into **6 focused batches of 4 modules each**, run each
with **claude-sonnet-4-6**, then merge into one `srs.md` for use in all downstream steps.

## Quick Start

### Prerequisites

```bash
pip install anthropic pyyaml rich
export ANTHROPIC_API_KEY="sk-ant-..."
```

### 1. Place your RBS

Copy your RBS file (Markdown or converted from Excel) to:
```
config/inputs/rbs.md
```
If your RBS is in Excel, convert it first:
```bash
python tools/convert_rbs_excel_to_md.py config/inputs/rbs.xlsx
```

### 2. Run all batches

```bash
python orchestrator_step_1_1.py
```

This will:
- Run 6 batches sequentially (each takes ~2–5 minutes on Sonnet)
- Save `srs-batch-N.md` after each batch
- Auto-merge into `srs.md` when all 6 complete

### 3. Check status at any time

```bash
python orchestrator_step_1_1.py --status
```

### 4. Run a single batch (if one failed)

```bash
python orchestrator_step_1_1.py --batch 3
```

### 5. Merge manually (if you ran batches separately)

```bash
python orchestrator_step_1_1.py --merge-only
```

### 6. Validate without re-running

```bash
python orchestrator_step_1_1.py --validate-only
```

## How the Prompt Works

The single `prompt.md` template is **resolved differently for each batch** by the orchestrator:

- `{{batch.number}}` → 1, 2, 3...
- `{{batch.module_names}}` → "Tenant & Auth Core, Academic Configuration, Staff Management, Student Management"
- `{{input:batch:rbs_extract}}` → Only the RBS rows for this batch's modules (orchestrator extracts this automatically)

This means Claude only sees 4 modules' RBS data at a time and only generates SRS for those 4 modules.
The full quality of the original prompt is preserved — no sections removed.

## Output: What Goes Where

After all batches complete and merge runs:

```
output/srs.md  →  feed as input to:
                    Step 1.2 (Dependency Map)
                    Step 1.3 (Data Dictionary)   [reads srs.md]
                    Step 1.4 (System Architecture) [reads srs.md]
```

The batch files (`srs-batch-N.md`) are kept as intermediate artefacts.
You can delete them after a successful merge to save space.

## Troubleshooting

**Batch output looks truncated (ends mid-sentence)**
→ The `min_length` validator will flag this. Re-run that batch:
```bash
python orchestrator_step_1_1.py --batch N --force
```

**A module's FRs are missing from the output**
→ Check if that module's name appears in your RBS. The orchestrator extracts RBS sections
by matching module names/codes. If your RBS uses different naming, update `batch_config.yaml`
module names to match your RBS headings exactly.

**Rate limit hit mid-run**
→ The orchestrator retries automatically with backoff. State is saved after each batch,
so a full restart picks up where it left off.

**Merged srs.md is missing some modules**
→ Run `--validate-only` to see which batches have issues, then re-run the failing batch.
