1.1 Module Purpose

The Syllabus module is the academic content backbone of the platform. It defines the hierarchical curriculum structure — from Class/Subject down to Lesson and multi-level Topic trees. All other LMS modules (Quiz, Quest, Exam, Homework, QuestionBank) depend on this module for curriculum context.

1.4 Topic Hierarchy Rules

  • Topics form a recursive tree via parent_id with automatic materialized path computation
  • Level 0 = root topic; computed as parent.level + 1 automatically
  • Code auto-generated: CLASS_SUBJECT_LESSON_TOP01, sub-topics: ..._SUB01
  • Level prefixes: TOP(0), SUB(1), MIN(2), SMT(3), MIC(4), NAN(5), ULT(6)
  • Topic name unique per (lesson + parent)

2.1 Module Purpose

Manages formative assessments in quiz format. Teachers create quizzes linked to an academic hierarchy (session → class → subject → lesson → topic), configure question settings, difficulty distribution, and allocate quizzes to classes, sections, groups, or individual students.

2.2 Quiz Lifecycle

DRAFT → PUBLISHED → ARCHIVED

No state machine enforces transitions. Student access: status = PUBLISHED AND is_active = true.

3.1 canPublish() Logic (Quest Model)

public function canPublish(): bool {
    if ($this->questQuestions()->count() === 0) return false;
    if ($this->questQuestions()->count() !== $this->total_questions) return false;
    if (!empty($this->validateSettings())) return false;
    if (!$this->academic_session_id || !$this->class_id ||
        !$this->subject_id || !$this->lesson_id) return false;
    return true;
}

Common Business Rules
  1. An entity can be soft-deleted and restored independently of its is_active state.
  2. is_active = false + deleted_at = null = deactivated but visible to admins.
  3. deleted_at != null = soft-deleted, accessible only via trash view.
  4. Academic session is the top-level scope for all teaching artifacts.
  5. Questions sourced from QuestionBank into assessments with ordinal and mark overrides.
  6. Availability flags control cross-module and cross-tenant access for questions.
  7. All allocations support multiple target types (CLASS/SECTION/GROUP/STUDENT).
  8. All FormRequest classes have authorize() → true; authorization done via controller $this->authorize() calls.
