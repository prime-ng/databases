## Phase 2A — Generate DDL SQL

### DDL Rules (MUST follow without exception):

**Table Standards:**
1. Table prefix: `{DB_TABLE_PREFIX}`
2. Every table MUST have these columns:
   - `id` — BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY
   - `is_active` — TINYINT(1) NOT NULL DEFAULT 1 COMMENT 'Soft visibility toggle'
   - `created_by` — BIGINT UNSIGNED NULL COMMENT 'FK to sys_users.id — audit'
   - `created_at` — TIMESTAMP NULL DEFAULT NULL
   - `updated_at` — TIMESTAMP NULL DEFAULT NULL
   - `deleted_at` — TIMESTAMP NULL DEFAULT NULL COMMENT 'Soft delete'
3. Use `IF NOT EXISTS` on all CREATE TABLE statements
4. Engine: InnoDB, Charset: utf8mb4, Collation: utf8mb4_unicode_ci

**Naming Standards:**
5. Junction tables suffix: `_jnt`
6. JSON columns suffix: `_json`
7. Boolean columns prefix: `is_` or `has_`
8. FK naming convention: `fk_{DB_TABLE_PREFIX}{table}_{column}`
9. Index naming: `idx_{DB_TABLE_PREFIX}{table}_{column}`
10. Unique naming: `uq_{DB_TABLE_PREFIX}{table}_{columns}`

**Data Type Standards:**
11. All IDs: `BIGINT UNSIGNED` (unless FK to existing table uses different type — match target)
12. All monetary values: `DECIMAL(15,2)` with COMMENT specifying currency
13. All percentages: `DECIMAL(5,2)`
14. All ENUM columns: use ENUM type with explicit values
15. Add `COMMENT` on EVERY column explaining its business meaning

**Integrity Standards:**
16. Index ALL foreign key columns
17. Add UNIQUE constraints where business rules demand it (from Feature Spec Section 3.1)
18. Add CHECK constraints for ENUM-like validations where MySQL supports them
19. All FKs must have explicit ON DELETE / ON UPDATE behavior

**Anti-Duplication Standards:**
20. Do NOT recreate tables that already exist in other modules
21. Do NOT add columns to existing tables from other modules (use ALTER TABLE in separate migration if needed, or create extension/config table)
22. Verify every cross-module FK target exists in the referenced DDL

### DDL Output Format:
```sql
-- ═══════════════════════════════════════════════════════════════

CREATE TABLE IF NOT EXISTS `{DB_TABLE_PREFIX}{table_name}` (
    `id`             BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    -- ... domain columns ...
    -- ... standard audit columns ...
    PRIMARY KEY (`id`),
```
