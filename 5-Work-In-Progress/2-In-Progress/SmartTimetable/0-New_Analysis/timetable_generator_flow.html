⚠ GAP ANALYSIS — 12 Issues
CRITICAL GAP-TT-001
Zero Authorization
SmartTimetableController has NO policy checks. Any authenticated user can trigger or overwrite generation.
CRITICAL GAP-TT-004
Sync HTTP Block
Runs synchronously on HTTP thread. Up to 300s. Must be queued job with polling endpoint.
HIGH GAP-TT-002
Parallel Types Missing
Only SAME_TIME implemented. CONSECUTIVE, ORDERED, SAME_DAY silently ignored — no error thrown.
HIGH GAP-TT-003
Silent Exceptions
Soft constraint exceptions swallowed. Returns score 0, silently skewing all placement decisions.
HIGH GAP-TT-005
Random Teacher Selection
Phase 2 uses array_rand — no workload balancing. Creates heavily uneven teacher loads.
HIGH GAP-TT-006
Silent Conflicts
Phase 3 force-placements: has_conflict=true in DB but NO admin notification or dashboard alert.
MEDIUM GAP-TT-007
Weak Diagnostics
Per-activity failure reason not recorded. Admins can't tell WHY an activity couldn't be placed.
MEDIUM GAP-TT-008
No Partial Regen
is_locked flag in tt_timetable_cells NOT honored — all locked cells overwritten each run.
MEDIUM GAP-TT-009
N+1 Risk
Teacher name cache built per-run. If not eagerly loaded, each miss triggers individual DB query.
MEDIUM GAP-TT-010
No Live Progress
No SSE/polling endpoint. User sees no feedback during 25–300s generation run.
LOW GAP-TT-011
God Controller (2,958 lines)
Should split into: GenerationController, ConstraintController, ActivityController, ViewerController.
LOW GAP-TT-012
Verbose Logging Lost
FETSolver log() calls only write to stdout in debug_mode. No Log::debug() — lost in production.
