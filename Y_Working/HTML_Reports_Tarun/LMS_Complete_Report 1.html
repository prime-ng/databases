Data Flow & Lifecycle

A. Quiz/Quest Attempt Flow
1
Teacher creates Quiz/Quest + links questions + allocates to students
Tables: lms_quizzes/lms_quests + lms_quiz_questions/lms_quest_questions + lms_quiz_allocations/lms_quest_allocations
2
Student sees allocation in "My Learning" and clicks "Start"
StudentPortal reads allocations, creates row in lms_quiz_quest_attempts (status=IN_PROGRESS)
3
Student answers questions one by one
Each answer saved to lms_quiz_quest_attempt_answers. Checkpoint saved to lms_attempt_checkpoints every 30s
4
Student submits OR timer expires (auto-submit)
Attempt status changes to SUBMITTED. MCQ answers auto-graded immediately. Descriptive answers marked is_evaluated=0
5
Teacher reviews descriptive answers (if any)
Teacher marks each descriptive answer, sets marks_obtained + evaluation_remarks
6
Result computed & published
Row created in lms_quiz_quest_results with total score, percentage, pass/fail. is_published=1 makes it visible to student
B. Exam Attempt Flow (Online)
1
Admin creates Exam + Papers + Sets + Questions + Allocates
Tables: lms_exams + lms_exam_papers + lms_exam_paper_sets + lms_paper_set_questions + lms_exam_allocations
2
Student starts online exam
Row created in lms_exam_attempts (attempt_mode=ONLINE, status=IN_PROGRESS). Full-screen mode activated
3
Student answers questions + proctoring events logged
Answers saved to lms_exam_attempt_answers. Violations (tab switch, focus loss) logged to lms_attempt_activity_logs
4
Auto-grading + Teacher evaluation
MCQ auto-graded. Descriptive answers queued for teacher review. Status=EVALUATION_PENDING
5
Result published
Row in lms_exam_results with total marks, percentage, grade, division, rank. Status=RESULT_PUBLISHED
C. Exam Flow (Offline — Upload, Check & Publish)
1
Same admin setup as online
Exam paper mode = OFFLINE. Questions and blueprints still defined in system for structured evaluation.
2
Exam conducted physically in classroom + attendance marked
Teacher marks students as PRESENT/ABSENT. System creates lms_exam_attempts (attempt_mode=OFFLINE, is_present_offline=1/0).
3
Teacher uploads scanned answer sheets
Teacher scans/photographs student answer sheets and uploads them (PDF or images). Each file linked to the student's ExamAttempt via offline_paper_uploaded_id (sys_media). Supports batch upload with drag & drop.
4
Teacher checks answer sheets online
Split-screen UI: LEFT = zoomable scanned sheet viewer (page navigation). RIGHT = question-wise marks entry panel. Teacher can annotate the sheet (tick, cross, circle, comments) using canvas overlay. Annotations saved as JSON. Alternatively, BULK_TOTAL mode for quick single-number entry.
5
Checked sheet generated
System composites original scanned pages + teacher annotations + marks summary into a final "checked" PDF. Stored for student access.
6
Result published + Student views checked sheet
ExamResult created with total marks, grade, division, rank. is_published=1. Student sees the checked answer sheet in their Results page — original handwriting with teacher's annotations, marks per question, and total. Downloadable as PDF. Parent can also view.
D. Homework Flow
1
Teacher creates homework + publishes
Bulk creates lms_homework_assignment rows for each student (status=PENDING)
2
Student sees homework in "My Learning" and submits
Creates row in lms_homework_submissions (file upload or text). Assignment status=SUBMITTED
3
Teacher reviews and grades
Teacher updates submission with marks + feedback. Assignment status=REVIEWED/GRADED
