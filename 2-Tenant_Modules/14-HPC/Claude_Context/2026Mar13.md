## DomPDF Critical Rules (Learned from Bug Analysis)

| ❌ Does Not Work in DomPDF | ✅ Use Instead |
|---|---|
| `display:flex` | `<table>` with `<td>` columns |
| `display:grid` | `<table>` layout |
| `gap`, `align-items`, `justify-content` | `cellspacing`/`padding` on `<td>` |
| `calc()` in any CSS property | Fixed `px` or `mm` values |
| `object-fit:cover` | Remove; use `width`/`height` on `<img>` |
| `box-shadow` | Remove entirely |
| `opacity` | Remove entirely |
| `z-index` | Remove; rethink layout |
| `position:absolute` with `%` coords | Use `<table>` for positioning |
| `cursor:default` | Remove |
| `asset('emoji/...')` URLs | `$pdfImg()` helper with `public_path()` + `base64_encode()` |
| Unicode emoji chars (📚📰🧩) | `<img>` tag via `$pdfImg()` helper |
| `min-height:267mm` per section | Remove; let content flow naturally |
| `<x-component>` Blade tags | Inline the component HTML directly |

### The `$pdfImg` Helper Pattern (must be at top of every PDF template)
```php
@php
$pdfImg = function(string $relativePath, int $w = 20, int $h = 20, string $extra = '') use (&$pdfImg): string {
    $fullPath = public_path($relativePath);
    if (!file_exists($fullPath)) return '';
    $ext  = strtolower(pathinfo($fullPath, PATHINFO_EXTENSION));
    $mime = $ext === 'svg' ? 'image/svg+xml' : "image/$ext";
    $b64  = base64_encode(file_get_contents($fullPath));
    return "<img src=\"data:{$mime};base64,{$b64}\" width=\"{$w}\" height=\"{$h}\" style=\"vertical-align:middle;{$extra}\">";
};
@endphp
```

## New Feature: Send Report Card via Email (Queue)

### Flow
```
Student list → "Send Email" button (per row, envelope icon)
  → AJAX POST to POST /hpc/send-report-email
  → HpcController::sendReportEmail() validates + dispatches Job
  → Returns JSON immediately: {success: true, message: "Queued for N guardian(s)"}

Queue worker (background):
  → SendHpcReportEmail::handle()
  → tenancy()->initialize($tenantId)    ← MUST re-init tenancy
  → Load student, template, saved values (same as generateSingleStudentPdf)
  → Render Blade view → DomPDF → save PDF to storage
  → Load guardians from std_guardians WHERE email IS NOT NULL
  → Mail::to($email)->send(new HpcReportMail(...))  ← attaches PDF
```

### Files to Create/Modify
| File | Action |
|------|--------|
| `Modules/Hpc/app/Jobs/SendHpcReportEmail.php` | CREATE — ShouldQueue job |
| `Modules/Hpc/app/Mail/HpcReportMail.php` | CREATE — Mailable with PDF attachment |
| `Modules/Hpc/resources/views/emails/hpc-report.blade.php` | CREATE — email body view |
| `HpcController.php` | ADD `sendReportEmail()` method |
| `HpcController.php` | CHANGE `buildPdf()` + `minifyHtml()` from `private` to `public` |
| `routes/tenant.php` | ADD `Route::post('/send-report-email', ...)` inside HPC group |
| `student-list/index.blade.php` | ADD envelope button per row + AJAX handler |

### Key Rules
- Job dispatches to `emails` queue: `->onQueue('emails')`
- MUST call `tenancy()->initialize($tenantId)` at start of `handle()` — queue workers run outside tenant context
- Pre-flight checks in controller: guardian emails exist, template resolved — return 422 with friendly message if not
- `Mail::to()->send()` inside the job (NOT `Mail::queue()`) — job itself provides queuing
- 3 retries, 5-minute timeout (PDF generation is slow)

### Queue Setup
```bash
QUEUE_CONNECTION=database  # in .env
php artisan queue:table && php artisan migrate
php artisan queue:work --queue=emails
```
