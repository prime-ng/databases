## Final Architecture — PDF Generation

### Flow
```
Student List Page
  → User selects students (checkboxes) + academic term
  → Clicks "Generate PDF" button
  → AJAX POST to route: hpc.generate-report
  → HpcController::generateReportPdf()
  → Renders hpc_form/pdf/report.blade.php (includes partials with isPdf=true)
  → DomPDF generates PDF
  → Saved to storage/app/public/hpc-reports/pdf/
  → URL returned → opened in new tab
```

### Route
```php
// routes/tenant.php (line ~2366)
Route::post('/generate-report', [HpcController::class, 'generateReportPdf'])->name('generate-report');
```

## HpcController.php — Key Functions

### `hpc_form(Request $request, $student_id = null)` — line ~282
- Web view for displaying/editing the HPC form
- Loads template, student data, siblings, attendance, saved values
- Returns appropriate blade view (first_form / second_form / thread_form / fourth_form)
- Side effect: upserts attendance data into `hpc_template_section_table`

### `generateReportPdf(Request $request)` — line ~1208 (REPLACED from generateReport)
- Validates `student_ids[]` + `academic_term_id`
- For each student:
  - Calls `resolveTemplateId()` to get template 1/2/3/4
  - Loads template (cached per templateId across students)
  - Fetches student data, siblings, attendance, saved values — **identical to hpc_form**
  - Renders `hpc::hpc_form.pdf.report` view
  - Uses DomPDF (`\Barryvdh\DomPDF\Facade\Pdf::loadHTML($html)->setPaper('a4', 'portrait')`)
  - Saves to `storage/app/public/hpc-reports/pdf/`
- Returns JSON: `{ success, pdf_url, pdf_urls[], message, warnings[] }`

### `resolveTemplateId(Student $student): ?int` — line ~1502
- Mirrors the template-ID logic from the student-list blade
- Uses `$class->ordinal`:
  - ordinal ≤ 0 → template 1 (Pre-primary)
  - 1–5 → template 2
  - 6–8 → template 3
  - 9–12 → template 4
- Falls back to class name parsing (keywords, numbers, Roman numerals)

### `formStore(Request $request)` — line ~644
- Saves/updates HPC report data submitted from the web form
- Handles all field types (text, boolean, json, table cells, images, etc.)

## mPDF Crash Root Cause (for reference — no longer relevant)

Error: `Trying to access array offset on false [Mpdf.php:16658]`

```php
// mPDF line 16658:
if ($this->tableLevel) {
    $this->SetLineHeight('', $this->table[1][1]['cellLineHeight']);
}
```

**Cause:** CSS `display: table` div containing `display: table-cell` divs **without** an intermediate `display: table-row`. mPDF requires explicit `table-row`; browsers auto-create anonymous rows — mPDF does not. The `$this->table[1][1]` is never initialised → crash.

**Affected CSS classes:**
- `.three-column-layout { display: table }` + `.left-column/.middle-column/.right-column { display: table-cell }` — no `display: table-row` wrapper
- `.pin-wrapper { display: table }` + `<input class="pin-box">` — after sanitisation inputs became bare text (no table-cell children)

**Fix (applied to pdf/first_form.blade.php, also incorporated in report.blade.php):**
- `display: table` → removed / changed to `display: block`
- `display: table-cell` on columns → changed to `display: inline-block`

## HpcReportService — Bug Fixed

**File:** `Modules/Hpc/app/Services/HpcReportService.php`

**Bug:** `json_decode($item->in_json_value, true)` returns PHP `false` (not `null`) when
the stored JSON is the literal string `"false"`. This caused `isset($savedValues[$key])`
in the blade view to trigger "Trying to access array offset on false".

**Fix:**
```php
$jsonVal = $item->in_json_value ? (json_decode($item->in_json_value, true) ?? null) : null;
if ($jsonVal === false) $jsonVal = null;
```

## Performance Optimisations Applied to HpcController (from Session 1)

1. **Template caching** — `$loadedTemplates[$templateId]` prevents re-fetching template for each student in bulk PDF
2. **Single HpcReportService instance** — created once, reused across all students
3. **Attendance: 24 queries → 2** — `selectRaw/groupBy` instead of per-month queries
4. **Attendance upsert: 96 queries → 1** — `HpcTemplateSectionTable::upsert()` with unique key `[section_id, row_id, column_id]`
5. **Illness total** — single full-year query, grouped by month in PHP
6. **tableCellMapping** — batched `whereIn` instead of N per-section queries
7. **globalRubricMapping** — scoped to current template's rubric IDs only
